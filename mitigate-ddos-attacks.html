Mitigate DDoS Attacks

Learn what DDoS attacks are, different types of DDoS attacks, why they are harmful to websites, how they work, and how to avoid them.

DDoS Attacks are harmful to websites. Learn how they work & what they do.

A DDoS Attack (short for distributed denial of service attack) is a type of cyber-attack which typically targets a website. During a DDoS attack, a large amount of requests are sent to the target website, in an attempt to use up all of the servers resources (memory, CPU, etc) to stop functioning properly, therefore making the website inaccessible to the public.

○ Layer 7 DDoS

A layer 7 DDoS is a type of DDoS attack that sends HTTP traffic to a website in order to consume resources and hamper a website's ability to deliver content to visitors. An HTTP flood attack utilizes what appear to be legitimate HTTP requests to attack a website. We will recommend three different services in order to mitigate such attacks, being DDoS-Guard, BotGuard, and Cloudflare. DDoS-Guard and Cloudflare Pro are most effective, but BotGuard will certainly mitigate medium sized attacks. Alternatively, if you wish to manually configure software on your hosting server, we recommend trying Fail2ban, Modsecurity, or Bunkerized Nginx Webserver (which includes Fail2ban and Modsecurity).

○ Layer 3/4 DDoS

A Layer 4 DDoS attack is often referred to as a SYN flood. A TCP connection is established in what is known as a 3-way handshake. The client sends a SYN packet to the target server, the server responds with a SYN ACK, and the client responds to that with an ACK. After the three-way handshake is complete, the TCP connection is considered established. Attackers can send SYN Floods in an attempt to stop one's network from functioning properly. Layer 3 attacks often utilize the ICMP protocol which can be stopped by disabling ICMP. Layer 4 attacks are often easy to mitigate as long as you choose the right server provider. OVH provides DDoS Protection with all of their servers and has a network capacity of over 11 TBPS.

○ Other Suggestions

It is highly recommended that you have a web application firewall (WAF), standard firewall, and an intrusion prevention software on your web server. We recommend using Modsecurity, Fail2ban, UFW (which comes preinstalled in Ubuntu), and some of the providers mentioned in the Layer 7 section seen above on this page such as BotGuard, DDoS-Guard, or Cloudflare for a web application firewall. Additionally, for security, you should 1. always change your SSH port (default is 22), 2. always use SSH key authentication rather than password authentication, and 3. disable root SSH login.
