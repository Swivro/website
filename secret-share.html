 Secret Share

Secret Share allows you to secretly share self-destructing important messages in a quick, simple, secure, and private way. We don't log any messages and can not know what the contents of your message are. Secret Share is based on open source software.

AES 256-bit encryption

Secret Share utilizes strong AES 256-bit encryption, as well as transport encryption. Each time you create a new secret, a random 256-bit AES key is created, as well as a random prune date (365-730 days) if the secret is not viewed in time.

Is Secret Share open source and/or decentralized?

While Secret Share's source code is not public, it is based on open source software (AndrewPaglusch/FlashPaper on GitHub). We would consider it decentralized as anyone around the globe can host their own instance of FlashPaper (the software which Secret Share uses), however, the Secret Share product/service itself is centralized, as it is hosted on one server.

How do I know you won't look at my secret messages?

Swivro cannot see the secret messages you type onto Secret Share, as it is a zero-knowledge secret-sharing software. All secrets are encrypted securely. Additionally, secrets are destroyed immediately after being viewed. Each secret has a randomly-selected prune date (365-730 days) so that if the secret is never viewed by anyone, it is destroyed to prevent someone from cracking the secret URL link via bruteforce.

How exactly does Secret Share work?

Secret Share is based on open source software written in PHP. When you submit a secret, a random AES-256 bit private key is created, along with a random 128-bit IV and 64-bit ID. The ID and AES key are then hashed with bcrypt, and the submitted text is now encrypted with AES-256-CHC using the AES key and random IV. The ID and AES key now join (known as k). A random prune date is selected (any time between 365 to 730 days). This means that if the secret is not viewed before the randomly-selected prune date, the secret will be destroyed to prevent someone from eventually cracking the secret URL via bruteforce. The ID, IV, bcrypt hash, ciphertext, and prune epoch are stored in the sqlite database. The k value is returned to the user viewing the secret in the one-time URL. More info can be seen on the FlashPaper README.MD file, explaining how secrets are retrieved.

What is a SearX Node?
×
SearX is an open source search engine written in Python. Anyone can host their own public instance of SearX, which anyone can use (we call those SearX Nodes). Our database consists of many SearX Nodes. When you submit a search query here, your query is sent to one of the SearX instances instances in our database, picked randomly. This improves your privacy and anonymity.
